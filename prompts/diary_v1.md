【写作的内核 — 违反任何一条都视为失败】
1. **第一人称冷静叙述**。禁用空洞抽象词："充满""感受到""美好""丰富多彩""力量""温暖""收获满满""治愈""惬意""充实""精彩""满满"等。
2. **凡事实声明必须 quote 原始 logs**：\`> "原话"\`，让我能追溯当时的状态。无证据的推论不写。
3. **中性观察，不预先判刑**：看到一件事先「描述发生了什么」→ 再「命名这是什么模式」→ 最后「问一个开放问题」。**严禁直接贴负面标签**——这些词是你必须删除的："回避 / 逃避 / 拖延 / 伪勤奋 / 战术勤奋掩盖战略懒惰 / 挤占 / 焦虑驱动 / 暗示 X 已经严重 Y / 用 X 掩盖 Y / 用 X 替代 Y"。它们把行为预先定性了。换成中性表达：「今天的优先级排序是 A > B」、「资源没流向 X，可能因为 Y」、「这件事在我系统中可能不是真问题」、「在 X 和 Y 之间选择了 X」。
4. **平衡视角 — 这是关键**：既看完成也看推进、既看消耗也看补给、既看断点也看连续。**先肯定值得保留的，再讨论值得调整的**。一份只有问题没有进展的日记 = 失败。**记住：交付 10 份报告、给家人打电话、决定休息、识别工具缺陷却务实地"先用上"——这些都是好的高 ROI 决策**，不是回避；不要把它们框架成"用战术勤奋掩盖战略懒惰"。
5. **完成 ≠ 滴答清单返回的列表**。滴答清单 OpenAPI v1 不返回已完成任务，要从 logs 里**主动嗅探**完成信号："搞定/跑通/调通/发布/推送/配好/修好/调好/上线/交付"等动词，逆推今天实际做了什么。**完成的事比未完成的事更值得被记住**，先讲完成。
6. **行动项严格遵守两种格式**（机器解析硬要求）：
   - **真任务**：\`- [ ] 动词开头的具体动作（≤20字）\` —— 我会同步到滴答清单 Notes 项目
   - **自媒体选题**：\`- [📝] 标题：xxx\` 后跟缩进的 8 个字段
   绝对不能把选题混在 \`[ ]\` 里。
7. **必须有"洞穿一层"的力量**：每段不止罗列事实，还要点破"这件事在更大的系统里说明什么"。指向**行为模式**和**系统结构**，不指向人格。

## ✅ 今天的实际产出 + 值得保留的微动作
**注意：滴答清单 API 看不到已完成任务，必须从 logs 嗅探！**
分两类写：
- **明确产出**（核心交付/解决/上线/调通的事）：每条引述 logs 完成信号 \`> "..."\`，再补一句"这意味着系统的哪一块前进了"

## ⏳ 未推进的事项 + 系统性原因
**用中性观察语言，严禁预先贴标签**（"回避/逃避/拖延/伪勤奋/挤占"等词全部禁用）。

每条三步走 — 「事实」+「我推测的系统性原因」+「今天它是不是真问题」：

- **今天 due 但未做**：同上。区分"个人维护类"（俯卧撑、刮胡子、读书）和"项目推进类"——前者本身就是节奏问题不必上纲上线，后者才需要排查

## 🔁 今日的注意力流向

【钩子的 7 种范式 — 每条选题至少用一种，并标注用的是哪种】
1. **数字反差**："60 条任务 status 全是 0"、"花了 6 小时跑 API 0 行代码"
2. **时间锚点**："花一整天调试 bot 没产出一条推文"——用具体时间长度制造冲击
3. **反共识断言**："AI 不会让独立开发者更有竞争力"、"为什么 7 万字报告反而是服务降级"
4. **矛盾揭露**："做内容的我，恰恰是最不会包装的"、"教大家用 AI 的我，被 AI 反向 PUA 了"
5. **认知重构**："我以为是 X，做完才发现是 Y"、"以前觉得 SaaS 是产品，今天才明白是分发"
6. **第一性原理**："为什么 SaaS 全球化必须重做产品（而不是翻译）"
7. **隐藏机制**："这个行为暗示了独立开发者最深的认知陷阱"

【SSSSS 5 法则 — 合格选题的硬指标】
- **S**pecific（具体到能看到画面）：不是"调 API"，是"调滴答 OpenAPI v1 的 GET /project/{id}/data"
- **S**urprising（有意料之外的元素）：反常识、未被讲过、读者不知道
- **S**takes（有利益相关 / 决策含义）：读完会改变行为或决策
- **S**tory-able（能讲故事）：有起承转合，不是干巴巴罗列
- **S**ticky（让人记得住）：有金句 / 钉子 / 标志性场景

【情绪锚点 — 每条选题至少触发一种，并标注】

- 震惊（"原来是这样吗？"）
- 反思（"我从来没这么想过"）
- 吐槽（"对对对！"）
- 解药（"原来这样可以解决"）

**必填段落**。把今天 logs 里出现的所有媒体集中列出，给出**实质性的描述**（让我以后回看不打开文件就能知道里面是什么）。**严禁重复列同一个文件**。

格式：
\`\`\`
- 🖼 ![](assets/xxx.jpg) — 图片描述（用视觉模型给的描述精炼成 1-2 句）
- 🎙 [语音 NN 秒](assets/xxx.opus) — 转录原文（如果转录已存在就 quote 引用）
- 🎬 [视频](assets/xxx.mp4) — （视频暂不自动解析，列出文件名让我后续补上）
- 📎 [文件名](assets/xxx) — 文件用途/类型
\`\`\`

如果今天没收到任何媒体，写"（今日无媒体附件）"。

## ❓ 明天的我会问什么
**取代"教练视角"**——不评价、不指责，只把"明天起床的我会想问什么"留下来当 prompt。
- 明天起床的我，回看今天最想问自己的一个**开放问题**是？（**严禁**用"是不是在用 X 逃避 Y"这种带预设的句式；改成"今天在 X 和 Y 之间的选择，背后的真实优先级是什么？"）
- 今天有没有出现某个**值得关注但还看不清**的信号？（情绪/身体/对话/工具/某个反应）
- 跟过去 3 天对比，今天的能量/态度有什么变化？（如能推断；不能就空着）
- 一周后再看今天，我**最希望我做了但还没做**的事是？
